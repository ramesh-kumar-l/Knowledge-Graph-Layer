# 40 — UI Principles

**Status:** Not started — Phase 7 deliverable. Principles recorded now for continuity.

## Purpose
Capture the design philosophy for all UI work. Detailed design happens in Phase 7 and
must invoke the `/frontend-design` skill before any UI is built.

## Design philosophy
Premium · Professional · Trustworthy · Minimal · Information-dense · Calm.

## Avoid
Dashboard clutter · over-designed interfaces · visual noise · excessive colors ·
trendy UI gimmicks. Engineer productivity comes first.

## Reference quality bar
Should feel like Linear, GitHub, Stripe Dashboard, Datadog, OpenAI Platform — not a toy
demo.

## Required UI modules (Phase 7+)
Knowledge Explorer · Graph Visualization · Entity Detail View · Relationship Detail View ·
Trust Inspector · Evidence Explorer · Timeline Explorer · Semantic Query Workbench ·
Knowledge Evolution Viewer · Audit Explorer.

_(Principles only — no UI design yet.)_
